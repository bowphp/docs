<p align="center">BARRY is the ORM (<strong>Object Relation Mapping</strong>) built into BowPHP.</p>

## Introduction

An ORM is a way of linking tables together using classes. Here, each record in a table represents an Object that can be related to other records. This is what is called **Object Relation Mapping** :wink:.

The BARRY ORM included with BowPHP provides a beautiful and simple ActiveRecord implementation for working with your database. Each database table has a corresponding "model" that is used to interact with that table. Models allow you to query for data in your tables, as well as insert new records into the table.

In every self-respecting framework, there is an ORM system with a super cute name. Bow's is called **BARRY**, isn't that cute :smile:.

> Before getting started, make sure you have configured a database connection in `config/database.php`.

Before continuing, please add a migration:

```bash
php bow add:migration CreateTodoTable
```

Then edit the migration:

```php
public function up()
{
  $this->create("todos", function (SQLGenerator $table) {
    $table->addIncrement('id');
    $table->addString('title');
    $table->addString('status', 0);
    $table->addInteger('budget', 0);
    $table->addTimestamps();
  });
}
```

Finally, run the migration:

```bash
php bow migration:migrate
```

> This migration will be used to allow you to run the tests directly on the `App\Models\Todo::class` model.

## Adding a model

To add a model, use the `php bow` command line with the `add:model` command followed by the model name.

```bash
php bow add:model Todo
```

After creating the model, a file with the same name will be created, in our case `Todo.php`, at the root of the `app/Models` folder.

Here is an overview of the file:

```php
namespace App\Models;

use Bow\Database\Barry\Model;

class Todo extends Model
{
  //
}
```

> Before using the model, make sure you have configured your database

### Table Name

Note that we did not tell BARRY which table to use for our Todo model. By convention, the "snake case", plural name of the class will be used as the table name unless another name is explicitly specified.

You can manually specify a table name by defining a table property on your model:

```php
namespace App\Models;

use Bow\Database\Barry\Model;

class Todo extends Model
{
  /**
   * Define the table associated with the model.
   *
   * @var string
   */
  protected $table = 'tbl_todos';
}
```

### Primary keys

BARRY will also assume that each table has a primary key column named id. You can define a protected `$primary_key` property to override this convention:

```php
namespace App\Models;

use Bow\Database\Barry\Model;

class Todo extends Model
{
  /**
   * The primary key associated with the table.
   *
   * @var string
   */
  protected $primary_key = 'id_todo';
}
```

## Retrieving data

Once you have created a model and its associated database table, you are ready to start retrieving data from your database. Think of each BARRY model as a powerful [query builder](./query-builder) allowing you to fluently query the database table associated with the model.

For example:

```php
use App\Models\Todo;

$todos = Todo::all();

foreach ($todos as $todo) {
  echo $todo->title;
}
```

The `find` and `findBy` methods also allow you to retrieve information:

```php
// With find
$todo = Todo::find(1);

// With findBy
$todo = Todo::findBy('id', 1);
```

> The method can also return `null` in the case where no record is found

## Adding additional constraints

The BARRY `all` method will return all of the results in the model's table. Since each BARRY model serves as a [query builder](./query-builder), you can also add constraints to queries, and then use the get method to retrieve the results:

```php
$flights = App\Models\Todo::where('status', 'done')
            ->orderBy('titie', 'desc')
            ->take(10)
            ->get();
```

## Retrieving aggregates

You can also use the `count`, `sum`, `max`, and other aggregate methods provided by the [query builder](./query-builder). These methods return the appropriate scalar value instead of a full model instance:

```php
use App\Models\Todo;

$count = Todo::where('status', 'done')->count();

$max = Todo::where('status', 'done')->max('budget');
```

## Inserting and updating models

### INSERT

To create a new record in the database, create a new model instance, set attributes on the model, then call the save method:

```php

namespace App\Http\Controllers;

use App\Controllers\Controller;
use App\Models\Todo;
use Bow\Http\Request;

class TodoController extends Controller
{
  /**
   * Create a new todo instance.
   *
   * @param Request $request
   * @return mixed
   */
  public function store(Request $request)
  {
    // Validate the request...

    $todo = new Todo;

    $todo->title = $request->get('title');
    $todo->budget = $request->get('budget', 0);
    $todo->status = 'pending';

    $todo->save();
  }
}
```

In this example, we assign the name parameter from the incoming HTTP request to the `title` and `budget` attributes of the `App\Models\Todo` model instance. When we call the `save` method, a record will be inserted into the database. The `created_at` and `updated_at` timestamps will automatically be set when the save method is called, so there is no need to set them manually.

### Insert via CREATE

Active Record objects can be created from a hash, from a block, or have their attributes set manually after creation. The new method will return a new object while `create` will return the object and save it to the database.

For example, given a model user with name and occupation attributes, calling the create method will create and save a new record to the database:

```php
use App\Models\Todo;

$user = Todo::create([
  'title' => 'Acheter un ticket metro',
  'budget' => 2000,
  'status' => 'pending',
]);
```

### UPDATE

The `save` method can also be used to update models that already exist in the database. To update a model, you should retrieve it, set the attributes you wish to update, then call the `save` method. Again, the `updated_at` timestamp will automatically be updated, so there is no need to manually set its value:

```php
use App\Models\Todo;

$todo = Todo::find(1);

$todo->title = 'Shopping pour Franck';

$todo->save();
```

Alternatively, you can also use the `update` method. You just need to define the conditions in order to limit the impact of the update.

```php
use App\Models\Todo;

Todo::where('status', 'done')
  ->update(['title' => 'Achat de ticket d\'avion']);
```

The `update` method expects an array of column and value pairs representing the columns to be updated.

## Deleting data

Likewise, once retrieved, an Active Record object can be destroyed, which removes it from the database.

```php
use App\Models\Todo;

$todo = Todo::find(1);
$todo->delete();
```

If you wish to delete multiple records in bulk, you can use the `destroyBy` or `truncate` method:

```php
// Delete all todo by id
Todo::deleteBy('id', 'David');

// delete all todo
Todo::truncate();
```
