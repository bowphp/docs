## Introduction

Sending emails in an application is a common task. BowPHP includes an easy-to-use email-sending system. You can use this system through several pilots commonly called drivers:

- **SMTP** - Direct sending via an SMTP server
- **SES** - You can also use [SES](https://aws.amazon.com/fr/ses/)
- **MAIL** - The system will use PHP's native `email` function

## Configuration

You will find the configuration in the `config/mail.php` file, which is relatively simple; all the options are commented.

Install the additional packages

- ses "aws/aws-sdk-php"

## Usage

To use the service, you must call the [`Bow\Mail\Mail::class`](https://bowphp.com/api/master/Bow/Mail) class. There are two static methods for sending mail: `email` and `raw_email`.

### The send method

**email** lets you send emails based on a [view](./views) in the application.

#### send prototype

```php
email($view, array $data, callable $callable)
```

| Parameter | Type |
|----------|------|
| view | `String` The recipient's email |
| data | `Array` The information to pass to the view |
| callable | `Closure` The message builder |

`$callable` lets you build the message and takes as a parameter an instance of [`Bow\Mail\Message`](https://bowphp.com/api/master/Bow/Mail/Message.html), which lets you add the recipient, the subject, the attachments, and the sender.

Example usage:

Consider the view `email-view.tintin.php`, which contains the following information:

```jinja
Bonjour {{ $name }},

Nous vous informons que vous compte viens d'être créditer de 100.000.000 F.

Cordialement,
```

Now let's write the code that sends the mail:

```php
use Bow\Mail\Mail;
use Bow\Mail\Message;

$data = [
  'name' => 'Franck',
];

Mail::send('email-view', $data, function (Message $message) {
  $message->to('info@exemple.com');
  $message->subject("Paiement !");
  $message->from("Bow <info@exemple.com>");
});
```

### Message API

BowPHP will therefore pass an instance of `Bow\Mail\Message`, which will let you add extra information to your email, notably the recipient.

##### `addHeader`, adding custom headers

```php
$message->addHeader(string $key, string $value)
```

##### `to`, sets the recipient

```php
$message->to(string $to, string $name = null)
```

##### `toList`, adding multiple recipients

```php
$message->toList(array $list_desc)
```

##### `addFile`, lets you add an attachment

```php
$message->addFile(string $file)
```

##### `subject`, sets the email subject

```php
$message->subject(string $subject)
```

##### `from`, sets the email sender

```php
$message->from(string $from, string $name = null)
```

##### `toHtml`, sets the content type to text/html

```php
$message->html($html)
```

##### `toText`, sets the message body

```php
$message->text(string $text)
```

##### `addBcc`, adds the Carbon Copy header

```php
$message->addBcc(string $mail, string $name = null)
```

##### `addCc`, adds the Carbon Copy header

```php
$message->addCc(string $mail, string $name = null)
```

##### `addReplyTo`, adds the Reply-To header

```php
$message->addReplyTo(string $mail, $name = null)
```

##### `addReturnPath`, adds the Return-Path header

```php
$message->addReturnPath(string $mail, $name = null)
```

### The raw method

**raw**, on the other hand, lets you send emails with a raw message.

#### raw prototype

```php
raw_email($to, $subject, $message, array $headers = [])
```

| Parameter | Type |
|----------|------|
| to | `String` The recipient's email |
| subject | `String` The email subject |
| message | `String` The message to send |
| headers | `Array` The headers to send, empty by default |

```php
use Bow\Mail\Mail;

$email = "hello@exemple.com";
$subject = "Hello, world";
$message = "C'est juste un email de teste";

Mail::raw($email, $subject, $message);
```
