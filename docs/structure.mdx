## Overview

BowPHP follows the **MVC** (Model-View-Controller) pattern, a proven architecture that clearly separates the responsibilities of your application.

```
bow-app/
├── app/                    # Application logic
│   ├── Configurations/     # Custom configurations
│   ├── Controllers/        # HTTP controllers
│   ├── Events/             # Event handlers
│   ├── Exceptions/         # Custom exceptions
│   ├── Listeners/          # Event listeners
│   ├── Middlewares/        # HTTP middlewares
│   ├── Models/             # Data models (ORM)
│   ├── Commands/           # Custom console commands
│   ├── Services/           # Business services
│   ├── Tasks/              # Asynchronous tasks (Queue)
│   ├── Validations/        # Validation rules
│   └── Kernel.php          # Kernel configuration
├── assets/                 # Frontend resources (sources)
│   ├── css/                # CSS files
│   ├── js/                 # JavaScript files
│   └── sass/               # SCSS files
├── config/                 # Configuration files
│   ├── app.php             # General configuration
│   ├── database.php        # Database configuration
│   ├── mail.php            # Email configuration
│   ├── storage.php         # Storage configuration
│   └── view.php            # View configuration
├── lang/                   # Translation files
│   ├── en/                 # English translations
│   └── fr/                 # French translations
├── migrations/             # Database migrations
├── public/                 # Web entry point (documentroot)
│   ├── index.php           # Front controller
│   ├── css/                # Compiled CSS
│   └── js/                 # Compiled JS
├── routes/                 # Route definitions
│   └── app.php             # Application routes
├── seeders/                # Data seeders
├── templates/              # Tintin views
│   ├── layouts/            # Shared layouts
│   └── partials/           # Reusable components
├── tests/                  # Unit and functional tests
├── var/                    # Generated files
│   ├── cache/              # Application cache
│   ├── logs/               # Error logs
│   ├── session/            # User sessions
│   ├── storage/            # Uploaded files
│   └── view/               # Compiled view cache
├── .env.json               # Environment variables
├── bow                     # Bow Console CLI
├── composer.json           # PHP dependencies
└── package.json            # Node.js dependencies
```

## The app/ folder

The heart of your application. This is where you write your business logic.

### Controllers/

Controllers handle HTTP requests and return responses.

```php title="app/Controllers/UserController.php"
namespace App\Controllers;

class UserController
{
  public function index()
  {
    return view("users.index");
  }
}
```

### Models/

Models represent your database tables.

```php title="app/Models/User.php"
namespace App\Models;

use Bow\Database\Barry\Model;

class User extends Model
{
  protected string $table = "users";
}
```

### Services/

Services encapsulate complex business logic. See the [services documentation](./service).

### Middlewares/

Middlewares filter HTTP requests. See the [middlewares documentation](./middleware).

### Events/ and Listeners/

An event system to decouple your application. See the [events documentation](./messaging).

### Tasks/

Asynchronous tasks executed in the background via the queue. See the [tasks documentation](./task).

### Kernel.php

Configures the application bootstrap:

```php title="app/Kernel.php"
namespace App;

use Bow\Configuration\Loader;

class Kernel extends Loader
{
  public function configurations(): array
  {
    return [
      // List of providers to load
      \Bow\Database\DatabaseConfiguration::class,
      \Bow\Mail\MailConfiguration::class,
    ];
  }

  public function middlewares(): array
  {
    return [
      "auth" => \App\Middlewares\AuthMiddleware::class,
    ];
  }
}
```

## The config/ folder

Contains all of the application's configuration files.

| File | Description |
|---------|-------------|
| `app.php` | General configuration (name, timezone, locale) |
| `database.php` | Database connections |
| `mail.php` | SMTP configuration and email drivers |
| `storage.php` | Storage disks (local, S3, FTP) |
| `view.php` | Template engine and cache |
| `session.php` | Session management |
| `cache.php` | Cache configuration |

## The routes/ folder

Defines the entry points of your API or web application.

```php title="routes/app.php"
$app->get("/", "HomeController::index");
$app->get("/users", "UserController::index");
$app->post("/users", "UserController::store");
$app->get("/users/:id", "UserController::show");
```

## The templates/ folder

Your Tintin views (or Twig/Blade depending on configuration).

```
templates/
├── layouts/
│   └── app.tintin.php      # Main layout
├── partials/
│   ├── header.tintin.php   # Header
│   └── footer.tintin.php   # Footer
├── home.tintin.php         # Home page
└── users/
    ├── index.tintin.php    # User list
    └── show.tintin.php     # User details
```

## The public/ folder

The only folder accessible from the web.

:::warning Security
Configure your web server to point to this folder only. Never expose the other folders.
:::

## The var/ folder

Files generated automatically by the application.

| Subfolder | Description |
|--------------|-------------|
| `cache/` | Application cache (config, routes) |
| `logs/` | Log files (errors, debug) |
| `session/` | Session files (if using the file driver) |
| `storage/` | Files uploaded by users |
| `view/` | Compiled views (cache) |

:::tip Permissions
Make sure the web server has write permissions on `var/`.
:::

## The .env.json file

Sensitive environment variables (not versioned):

```json title=".env.json"
{
  "APP_NAME": "Mon Application",
  "APP_ENV": "development",
  "APP_DEBUG": true,
  "APP_URL": "http://localhost:8080",
  "DB_CONNECTION": "mysql",
  "DB_HOST": "127.0.0.1",
  "DB_DATABASE": "bow_app",
  "DB_USERNAME": "root",
  "DB_PASSWORD": ""
}
```

## Best practices

:::info Code organization
- **One file = one class**: Follow the PSR-4 conventions.
- **Explicit naming**: `UserController`, `PaymentService`, `OrderCreatedEvent`.
- **Separation of concerns**: Keep controllers thin, put logic in services.
- **Centralized configuration**: Use `config/` and `.env.json`.
:::
